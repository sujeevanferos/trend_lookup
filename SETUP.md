## üì• Installation Steps

### Step 1: Clone the Repository
```bash
git clone <repository-url>
cd evolveXr2
```

### Step 2: Set Up API Keys

Create the API key files from the templates:

```bash
# Copy templates
cp resources/headlines/yt_key.example.py resources/headlines/yt_key.py
cp resources/weather/weather_key.example.py resources/weather/weather_key.py
```

Then edit each file and replace the placeholder with your actual API key:

**resources/headlines/yt_key.py:**
```python
API_KEY = "your_actual_youtube_api_key_here"
```

**resources/weather/weather_key.py:**
```python
API_KEY = "your_actual_openweather_api_key_here"
```

### Step 3: Set Up Python Environment

```bash
# Create virtual environment
python3 -m venv .venv

# Activate it
source .venv/bin/activate  # On Windows: .venv\Scripts\activate

# Install Python dependencies
pip install -r requirements.txt
```

### Step 4: Install UI Dependencies

```bash
cd ui
npm install
cd ..
```

### Step 5: Train ML Models (First Time Only)

The system uses two ML models that need to be trained:

```bash
# Train both models (takes ~5-10 minutes)
make train

# Or train individually:
make train-cat  # Categorization model
make train-opp  # Opportunity scoring model
```

**Note:** The models will be saved in:
- `engine/categorization_model/`
- `engine/opportunity_model/`

These directories are gitignored but will be created during training.

### Step 6: Create Output Directories

```bash
mkdir -p output jsons history
```

## üöÄ Running the System

### Option A: Quick Start (Development)

1. **Collect data:**
   ```bash
   python3 collect_all.py
   ```

2. **Process with ML pipeline:**
   ```bash
   python3 pipeline.py
   ```

3. **Start the API server** (in a separate terminal):
   ```bash
   python3 server.py
   ```

4. **Start the UI** (in another terminal):
   ```bash
   cd ui
   npm run dev
   ```

5. **Access the dashboard:** http://localhost:5173

### Option B: Docker (Production)

If you prefer Docker:

```bash
# Make sure API key files are created first!
docker-compose up -d

# Access API: http://localhost:8000
```

## üìÅ What's in the Repository vs What You'll Create

### ‚úÖ Tracked in Git (you'll get these):

- **node_modules/** - Installed via `npm install`
- **Output files** (`.json`, `.jsonl`) - Generated when you run the pipeline
- **Log files** - Generated during operation
- **Python cache** (`__pycache__`, `.venv/`)

## üêõ Common Issues

### "No such file or directory: yt_key.py"
‚ûú You forgot to create the API key files. See Step 2.

### "ModuleNotFoundError"
‚ûú Activate the virtual environment: `source .venv/bin/activate`

### "Model directory not found"
‚ûú Train the models first: `make train`

### UI shows "Failed to fetch"
‚ûú Make sure `server.py` is running on port 8000

### Docker: "API keys not found"
‚ûú Create the API key files BEFORE running `docker-compose up`
